Sitting at a cafe coding today, it struck me that Hugo's content types are a lot like database table design — each type has fixed fields (front matter), with independent list and detail views (list.html / single.html). MVC frameworks spent years abstracting this pattern, yet static site generators achieve something similar using just the filesystem. Elegant.
