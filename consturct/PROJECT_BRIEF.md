## 三、内容类型（共 6 种）

### 1. 文章（Posts）
- **定义**：个人思考与观点的完整输出
- **篇幅**：长（1000-5000 字）
- **更新频率**：低（周/月更）
- **字段**：标题、发布时间、正文、分类/标签
- **Hugo 实现**：`content/posts/` 下的 `.md` 文件

### 2. Podcast
- **定义**：用户自己的音频播客内容
- **核心介质**：音频
- **交互方式**：播放/订阅（不离开网站）
- **字段**：标题、发布日期、音频文件链接、简介、时长
- **Hugo 实现**：`content/podcast/` 下的 Markdown + 音频文件路径

### 3. Album（作品集/相册）
- **定义**：视觉作品集合（摄影、设计、艺术等）
- **核心介质**：图像
- **交互方式**：浏览/放大
- **字段**：作品标题、创作日期、图片（可多张）、描述
- **Hugo 实现**：`content/album/` 下的 Markdown + 图片目录

### 4. 零碎想法（Notes）
- **定义**：短内容、随笔、灵感快照（类似微博/推文）
- **篇幅**：短（50-500 字）
- **更新频率**：高（日/周更）
- **与文章的区别**：文章是完整思考，零碎想法是快速记录
- **字段**：发布时间、正文（可能带图片）
- **Hugo 实现**：`content/notes/` 下的 Markdown

### 5. 社交媒体链接
- **定义**：引导用户访问外部平台的内容（B站、YouTube、GitHub、Steam 等）
- **核心介质**：外部链接
- **交互方式**：点击跳转（离开网站）
- **关键设计要求**：❌ 不用图标式链接 ✅ 用内容预览驱动点击
- **字段**：平台类型、预览图片、标题、跳转链接
- **Hugo 实现**：数据驱动 `data/social-cards.yaml` 或单独内容类型

### 6. 好物分享（Shares）
- **定义**：推荐值得分享的东西（书、文章、商品、工具等）
- **篇幅**：中等（100-500 字）
- **更新频率**：中等
- **与文章的区别**：文章 = 我的思考总结；好物分享 = 我推荐的东西（外部价值）
- **字段**：推荐日期、物品名称、类型、推荐理由、链接、图片
- **Hugo 实现**：`content/shares/` 下的 Markdown

## 五、视觉/交互偏好

### 社交媒体链接区域
- 每个外部内容以「卡片」或「封面图」形式展示
- B站视频：视频封面 + 标题 → 点击跳转B站
- Steam 游戏：游戏封面图 → 点击跳转Steam
- 目标：用户看到「内容预览」而非「平台 logo」，提升点击率

### 好物分享区域
- 类似「荐物」/「值得买」的卡片风格
- 每个推荐：物品图片 + 名称 + 简短推荐理由 + 链接

### Album 区域
- 网格布局，图片为主
- 点击图片可放大或进入详情页
- 支持按项目/系列分组（可选）

### Podcast 区域
- 每条带 HTML5 audio 播放器或嵌入第三方
- 显示标题、简介、时长、发布日期
- 按时间倒序排列

## 七、已确认的设计决策

### 1. 各内容类型的页面结构

| 内容类型 | 列表页 | 详情页 | 说明 |
|----------|--------|--------|------|
| 文章 | ✅ | ✅ | 标准博客模式，列表浏览，详情阅读全文+评论 |
| Podcast | ✅ 单页 | ❌ | 列表中直接嵌入播放器，点击展开文字描述；无需跳转 |
| Album | ✅ | ✅ | 列表用网格展示缩略图，详情页看完整作品集/大图 |
| 零碎想法 | ✅ | ✅ | 列表作时间线流式展示，详情页用于独立分享/链接 |
| 好物分享 | ✅ | ✅ | 列表浏览推荐清单，详情页写完整推荐理由+链接 |
| 社交媒体链接 | ✅ 单页 | ❌ | 一个集合页展示所有外部内容卡片，点击直接跳转外站 |

### 2. Hugo 主题：从零开发

- 从零开发难度可控——Hugo 的 Go Template + HTML/CSS/JS，不涉及复杂框架
- 完全掌控页面结构和性能，避免主题臃肿
- 6 种内容类型各有独特布局，现成主题反而需要大量覆盖，不如直接写

### 3. 多语言支持：需要

- 使用 Hugo 内置 i18n 机制
- 内容文件按语言后缀区分（`content/posts/hello.zh.md` / `content/posts/hello.en.md`）
- 目标语言：中文（默认）+ 英文

### 4. 评论系统：Giscus

- 基于 GitHub Discussions，免费、无广告、无追踪
- 数据存储在用户自己的 GitHub 仓库，完全自有
- 无需自建后端，零维护成本（与项目目标一致）
